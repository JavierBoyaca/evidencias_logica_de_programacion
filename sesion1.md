  1.	¿Qué es la lógica de programación?
 La lógica de programación es la habilidad de pensar de manera clara y estructurada al escribir el código, siguiendo un conjunto de reglas y conceptos. Es la forma de organizar y planificar las instrucciones en un algoritmo, para que la computadora pueda interpretarlo y ejecutarlo sin errores. La lógica de programación se puede aplicar a diferentes paradigmas de programación, como el lógico, que usa los circuitos lógicos.
2.	¿Cuáles son los elementos básicos de la lógica de programación? 
Los elementos de la lógica de programación son:
1.	Instrucciones.
2.	Datos.
3.	Variables.
4.	Constantes.
5.	Operadores.
Además, la lógica de programación se compone de tres elementos primordiales: creatividad, lógica y razonamiento2.
3.	¿Cuál es la diferencia entre un lenguaje de programación y un lenguaje de marcado?
La principal diferencia entre un lenguaje de programación y uno de marcado es que el primero puede realizar operaciones aritméticas, lógicas y de control, mientras que el segundo solo puede definir la presentación o el formato de los datos
4.	¿Qué es un algoritmo y para qué se utiliza en la programación?
 Un algoritmo es un conjunto de instrucciones que se siguen para resolver un problema o realizar una tarea. Se utilizan en la programación para crear programas que ejecutan las instrucciones en una computadora o aplicación
5.	¿Cuáles son las principales diferencias entre los diferentes tipos de lenguajes de programación?
Normalmente se distingue entre los siguientes tipos de lenguaje de            programación:
•	Lenguajes de bajo nivel: diseñados para un hardware específico y que por lo tanto no pueden migrar o exportarse a otros computadores.
•	Lenguajes de alto nivel: más fáciles de entender y escribir que los lenguajes de bajo nivel.
•	Lenguajes de nivel medio: combinan características de los lenguajes de bajo y alto nivel2.Dentro de los lenguajes de alto nivel, también encontramos dos subtipos más, los lenguajes interpretables y los compilados.

6.	¿Cómo se utiliza la lógica booleana en la programación? 
La lógica booleana es un sistema basado en la lógica matemática que usa valores de verdadero o falso para crear reglas o expresiones lógicas1. Se utiliza en la programación para controlar el flujo de trabajo en el programa, mediante condicionales, operadores y funciones234. Por ejemplo, si x es verdadero y y es falso, entonces x AND y es falso.

7.	¿Qué son las estructuras de control de flujo y cómo se utilizan en la programación?
 Las estructuras de control de flujo son el conjunto de reglas que permiten controlar el orden en que se ejecutan las instrucciones de un programa. Hay tres tipos principales de estructuras de control: secuenciales, selectivas y repetitivas. Las estructuras secuenciales siguen el orden en que se escriben las instrucciones. Las estructuras selectivas ejecutan o no una instrucción dependiendo de una condición. Las estructuras repetitivas ejecutan una instrucción varias veces hasta que se cumpla una condición. Estas estructuras son la base de la programación estructurada y se encuentran en la mayoría de los lenguajes de programación.
8.	¿Qué es una variable y para qué se utiliza en la programación? 
Una variable en programación es un elemento que sirve para guardar, asignar y representar valores en una expresión de programación. Es una forma de almacenar datos en la memoria de la computadora, usando un nombre simbólico que se puede cambiar durante la ejecución de un programa. Las variables facilitan la tarea del programador y se pueden clasificar en diferentes tipos según el tipo de datos que contienen. 
9.	¿Cómo se utilizan las funciones en la programación? 
. Las funciones en programación son bloques de código que realizan una tarea específica y pueden ser reutilizadas en diferentes partes del programa. Las funciones tienen múltiples usos y beneficios, como la reutilización de código, la abstracción y la modularidad

10.	¿Qué es la programación orientada a objetos y cómo se utiliza en la programación? La programación orientada a objetos se basa en los siguientes principios:
•	Encapsulamiento: toda la información importante está contenida dentro de un objeto y solo se expone información seleccionada.

•	Abstracción: los objetos solo revelan mecanismos internos que son relevantes para el uso de otros objetos, ocultando cualquier código de implementación innecesario.


•	Herencia: las clases pueden reutilizar código de otras clases.
En la programación orientada a objetos, un objeto es una entidad que contiene datos y métodos que actúan sobre dichos datos. La programación orientada a objetos disminuye los errores y promociona la reutilización del código

}
